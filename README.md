# UEModManager

UEModManager 是一个基于 Electron 的桌面 Mod 管理器，**仅用于管理由虚幻引擎（Unreal Engine）开发的游戏 Mod**。  
不支持非虚幻引擎游戏。

## 应用定位

- 管理对象：UE 游戏的本地 Mod 文件夹、启用状态、加载顺序、标签、分组与预设
- 主要场景：本地 Mod 库管理、冲突规避、批量切换配置、Nexus 下载后安装
- 运行平台：Windows 10/11

---

## 功能总览

1. 多游戏管理（仅 UE 游戏）
2. Mod 导入（对话框导入 + 拖拽导入）
3. Mod 启用/禁用（支持主 Mod、子 Mod）
4. 同类组冲突管理（同组激活自动互斥）
5. Mod 重命名（文件名与显示名）
6. 优先级与加载顺序管理（优先级 + 拖拽排序）
7. 标签系统（筛选 + 批量编辑）
8. 预设系统（保存/编辑/删除/一键应用）
9. 预览图系统（悬停预览 + 画廊 + 上传/删除）
10. Nexus 内置浏览与下载后安装
11. 背景图与界面设置（主题、透明度、模糊、语言）
12. 快捷键与撤销（Undo）

## 功能与操作说明

### 1) 游戏选择与管理

#### 支持能力

- 游戏卡片选择进入
- 新增游戏配置
- 编辑游戏名称、描述、封面
- 网格/列表布局切换、卡片缩放

#### 如何操作

1. 启动应用后先进入“游戏选择页”。
2. **左键单击游戏卡片**：进入该游戏的 Mod 管理页。
3. **右键游戏卡片**：打开编辑弹窗，可修改名称、描述、封面。
4. 点击“添加游戏”：
   - 必填：游戏名称
   - 可选：游戏 ID、可执行文件名、封面图
   - 保存后会生成 `data/game/<gameId>.json` 配置。

---

### 2) 首次使用必做配置（路径）

#### 支持能力

- 设置 Mod 库目录
- 设置游戏根目录
- 设置 Nexus 下载目录
- 自动检测游戏路径（支持常见 Steam 路径场景）

#### 如何操作

1. 进入主界面后点击侧栏“设置”。
2. 在“路径”页填写：
   - `mods_dir`：你的 Mod 仓库目录
   - `game_path`：游戏根目录
   - `nexus_download_dir`：Nexus 下载目录（可选）
3. 可点击“自动检测游戏路径”尝试自动填充。
4. 修改后会自动保存；路径无效时应用会提示你重新设置。

---

### 3) Mod 导入（本地）

#### 支持能力

- 通过按钮导入文件夹
- 直接拖拽文件夹到窗口导入
- 自动跳过已存在同名 Mod

#### 如何操作

1. 点击侧栏“添加 Mod”，选择要导入的文件夹。
2. 或者直接把文件夹拖进主窗口。
3. 导入完成后列表会刷新，并提示“成功添加/跳过”数量。

说明：

- 导入逻辑按目录结构提取可用 Mod 文件夹。
- 同名目录已存在时会跳过，不会覆盖。

---

### 4) Mod 下载后安装（Nexus）

#### 支持能力

- 内置 Nexus 页面浏览与下载
- 下载进度显示
- 压缩包结构预览
- 选择要安装的目录项
- 支持 `zip / 7z / rar`

#### 如何操作

1. 在设置里先配置好下载目录（建议）。
2. 点击侧栏“Nexus Mods”打开内置浏览器。
3. 下载完成后，会弹出“安装选项”窗口。
4. 在文件树中勾选要安装的目录（含 `.pak` 且不再包含子目录的目录）。
5. 选择是否“安装为新 Mod 组”，可自定义 Mod 名称。
6. 点击“Install”完成安装。

---

### 5) Mod 启用/禁用

#### 支持能力

- 主 Mod 一键启用/禁用
- 子 Mod 独立启用/禁用（父 Mod 激活后可操作）
- 文件冲突提示与强制激活

#### 如何操作

1. 在 Mod 列表点击状态按钮（启用/禁用）。
2. 子 Mod 需要先展开父 Mod，再点击子 Mod 状态按钮。
3. 如果出现文件冲突弹窗，可：
   - 取消
   - 强制激活（覆盖冲突）

---

### 6) 加载顺序与优先级

#### 支持能力

- 主列表拖拽排序
- 子 Mod 拖拽排序
- 右键设置优先级（0~9）
- 拖拽排序锁（防误操作）

#### 如何操作

1. 将排序方式保持为“自定义”。
2. 使用拖拽调整主 Mod 或子 Mod 顺序。
3. 右键某个主 Mod，选择优先级（0~9）。
4. 可使用顶部“拖拽锁”按钮锁定排序能力。

---

### 7) 标签与筛选

#### 支持能力

- 搜索（名称/标签）
- 状态筛选（全部/已启用/未启用）
- 标签筛选（单选/多选）
- 批量标签编辑（添加/移除）

#### 如何操作

1. 顶部搜索框输入关键词过滤列表。
2. 点击“已启用/未启用”切换状态过滤。
3. 点击标签按钮进行标签过滤。
4. 多选 Mod 后点击“编辑标签”进行批量标签操作。

---

### 8) 子 Mod 关系管理

#### 支持能力

- 为主 Mod 添加子 Mod 关系
- 子 Mod 独立排序与启用状态
- 关闭子 Mod 时智能回滚覆盖文件

#### 如何操作

1. 先选中 1 个主 Mod。
2. 点击侧栏“添加子模组”。
3. 在弹窗中勾选要挂载的子 Mod，确认保存。
4. 回到列表展开该主 Mod，即可单独管理子 Mod。

---

### 9) 同类组（互斥组）管理

#### 支持能力

- 创建同类组（组内互斥）
- 向组中增删 Mod
- 重命名/删除组
- 激活冲突时自动停用同组其它成员

#### 如何操作

1. 多选 2 个及以上 Mod，点击“同类 Mod 管理”。
2. 输入组名并创建。
3. 在管理弹窗内可重命名组、向组中加 Mod、删除组、移除成员。
4. 组内成员激活时会自动处理互斥冲突。

---

### 10) 预设系统

#### 支持能力

- 保存当前启用组合为预设
- 编辑预设名称与颜色
- 删除预设
- 一键应用预设（批量切换主/子 Mod）

#### 如何操作

1. 点击侧栏“预设方案”打开预设面板。
2. 点击“保存当前方案”为新预设命名并保存。
3. 选中任意预设后可查看详情、编辑或删除。
4. 点击“应用预设”执行批量切换。

---

### 11) 预览图与服装图

#### 支持能力

- 悬停 Mod 卡片自动预览图轮播
- 点击上下文菜单打开大图画廊
- 为 Mod 上传/删除预览图
- 服装图库检索并反向定位相关 Mod

#### 如何操作

1. 鼠标悬停 Mod 行：右侧显示预览图。
2. 右键 Mod -> 预览 / 编辑预览图。
3. 在编辑预览图窗口中可上传新图或删除旧图。
4. 侧栏“服装列表”可检索并快速过滤对应 Mod。

---

### 12) 背景与界面设置

#### 支持能力

- 背景图片上传/删除/切换
- 背景透明度与模糊调节
- 主题切换
- 预览交互参数调整（延迟、轮播间隔）
- 拖拽边缘滚动触发距离设置
- 语言切换（`zh-CN / en / ja`）

#### 如何操作

1. 打开“设置”。
2. 在“外观”页配置背景、透明度、模糊、主题。
3. 在“交互”页配置预览与滚动参数。
4. 在“语言”页切换界面语言。

---

### 13) 目录与启动快捷操作

#### 支持能力

- 打开 Mod 仓库目录
- 打开游戏激活 Mod 目录
- 打开 Nexus 下载目录
- 一键启动游戏
- 切回游戏选择页

#### 如何操作

1. 使用侧栏“打开文件夹”下拉菜单进入不同目录。
2. 点击“启动游戏”直接运行配置的可执行文件。
3. 点击“切换游戏”返回游戏选择页。

---

### 14) 快捷键

- `Ctrl + F`：聚焦搜索框
- `Ctrl + A`：全选当前可见主 Mod
- `Shift + 点击`：范围多选
- `Delete`：删除选中项（含确认）
- `Space`：切换单个选中 Mod 的启用状态
- `Ctrl + Z`：撤销上一操作（Undo）
- `Esc`：关闭弹窗或清除选择

## 技术栈

- Electron 28
- Node.js + npm
- SQLite（`better-sqlite3`）
- `fs-extra` / `adm-zip` / `node-7z` / `node-unrar-js`

## 环境要求

- Windows 10/11
- Node.js 18+
- npm 9+

## 快速开始

```bash
npm install
npm start
```

## 打包发布

```bash
npm run dist
```

默认输出目录：`dist/`

## 项目结构

```text
src/main/         Electron 主进程与 IPC
src/renderer/     渲染进程页面、样式与前端逻辑
scripts/          工程脚本（迁移、构建备份等）
data/             运行时数据（数据库、游戏配置、封面、预设等）
dist/             打包输出
```

## 数据与配置

- 主数据库：`data/mod_manager.db`
- 游戏配置：`data/game/*.json`
- 游戏封面：`data/game_cover/`
- 预设目录：`data/preset/<gameId>/`

## 常用脚本

```bash
npm start
npm run pack
npm run dist
npm run migrate
npm run build:backup
```
